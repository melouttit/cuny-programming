jQuery Plugins

  • A common set of jQuery functions that can be added to your site for quick functionality
  • Anyone can write a plugin and release it for widespread use. Most will include documentation on how to implement and what custimzation is available to you
  • The jQuery plugin registry is a good starting point for seeing what is available.

Adding a Plugin

Download the Files

  • Download the plugin code from the source. On the homepage for your plugin, look for a download link.
  • Usually plugins will come with at least one JavaScript file, a CSS file and sometimes an images directory and README file.
  • Read the instructions for installing carefully. For most plugins, you will just include the source JS and CSS files on the page you want to instantiate the plugin on.

Minified JS

You might see two JS files in the plugin you download with filenames that are similar except for a -min or .min at the end. This means the owner is providing you with two ways to include their JS. The file with the min is the minified version, meaning it's stripped of white space and lighter to download. You would only want the unminified version if you were planning on altering the plugin.

Documentation

  • Look either in the plugins README file or on the website to find the documentation for how to use the plugin.
  • Most plugins require a standard HTML setup and then only a few lines of JS to get them working.
  • Look in the css directory for any style themes the plugin creator might provide.

Instantiation

  • The method that initiates the plugin in your code is usually the name of the plugin.
  • EXAMPLE: $(document).ready(function() {
      $('#carousel').slick();
    });
  • Where #carousel is the HTML element you want to instantiate the plugin on and slick() is the plugin method that runs the code you need.

Options

  • Beyond the standard setup, most plugins have options you can pass in to the customize the plugin.
  • You will need to read through the plugins documentation to find the options that are available to you.
  • The options are a set of properties, passed into the method inside curly brackets.

Options Example

  • $("#carousel").slick({
      autoplay: true,
      autoplaySpeed: 400
    });
  • Where autoplay: true is an option the plugin provides, allowing you to make the carousel loop through without clicking and autoplaySpeed sets the speed in which it loops.

Callbacks or Events

jQuery utilizes callback functions, which are found in plugins. These allow you to run a function when a specific event happens in the plugin.

Custom Methods

Additional customization options might come in the form of methods, where you can tap into the plugins internal methods to change how it works.

Demos

Many plugins also provide demo pages or have demos working on their page. Use the inspector to see how they set up the demos as a guide to setting up your own.

Carousel Plugin Exercise

Start a new branch of your website to practice with your plugins. We will use the images you've gathered to create a photo carousle with the slick plugin.

Create a Branch

In iTerm, navigate to your website and use git branch carousel to create a new branch to test your plugin on. Use checkout to switch to that branch.

Download the Plugin Files

Go to the slick plugin website and find the download link to grab the files from. Make sure to save the files in your website directory.

Find the Files

Go to the folder you just unzipped and, using the documentation as a guide, find the files you need.

Include Files

On your carousel page include the css and js files you need. Make sure to point to the right location.

Structure HTML

Write the HTML you need on your page, again using the documentation as a guide. Make sure the images you want are stacked on the page.

Instantiate

In the last set of script tags write the ready method and inside that instantiate your plugin with the slick method.

Options

Look through the options and test a couple of them on your plugin.

Commit Your Test
