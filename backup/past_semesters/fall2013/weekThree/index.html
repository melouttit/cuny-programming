What is CSS?

  • CSS stands for Cascading Style Sheets
  • It allows you to create rules that specify how elements on a website should look

Where do I put it?

  • Always in the head of your page
  • Inline Styles go inside the style tag 
    <style type="text/css"></style>
  • External Stylesheet are pulled in using the link tag
    <link rel="stylesheet" href="css/styles.css" />

Style Syntax

  • There are 2 parts to each CSS rule:
    • Selector - Indicates which element(s) the rule applies to. Can be tagnames, classes and IDs
    • Declaration - How the element referred to should be styled. There are two parts to each delcaration:
      • Property - The aspect of the element you want to style
      • Value - The setting you chooose for each property

Types of Selectors

  • Universal targets all elements on a page: *
  • Type matches an element name: h1, p, a
  • Class matches an element with that class value: .credit
  • ID matches the element with that ID value: #container
  • Child matches an element that is the direct child of another: li > a
  • Descendant matches an element that is a descendant of anotehr element (not necessarily a direct child): p a

How CSS Cascades

  • If two or more rules apply to the same element the one that takes precedence is either:
    • Last - If the selectors are identical the latter will take precedence. The stylesheet is read by the browser from top to bottom.
    • Specificity - If one selector is more specific it will take precedence.
  • Child elements, unless specificed in the styles, will usually inherit a style from their parent element. You can also delcare inherit as a value in your declarations.

EXAMPLES

Colors

  • You can specify the color of text, background and borders
  • There are currently 147 predefined colors that recognize by browsers, including black and DarkCyan
  • You can also use HEX codes and RGB values but you will most commonly come across HEX codes
  • You can grab these codes from Photoshop or one of the many charts that exist online, like this one.

How to pick colors

  • When picking foreground (text) and background colors, you want to make sure that there is enough contrast to be legible
  • Low contrast can make text difficult to read
  • High contrast makes text easier to read, but is better for shorter and important text, like headlines
  • Medium contrast is better for big blocks of text because it is easier on the eyes when viewed on a screen, and also for less important text, like image credits
  • Reverse, high contrast (like white text on a black background) should not be used for large blocks of text because it strains the eye. Example.

Text Terms

  • Serif fronts have extra details (serifs) on the ends of the main strokes of the letters
  • Sans-Serif have straight ends to letters
  • Monospace means each letter has the same width

Font Aspects

  • Weight - Light, Medium, Bold, Black
  • Style - Normal, Italic, Oblique
  • Stretch - Condensed, Regular, Extended

Adding Font Styles

  • font-family (Guide to predefined web fonts)
  • font-size
  • font-weight
  • font-style
  • text-transform
  • text-decoration
  • line-height
  • letter-spacing & word-spacing
  • text-align & text-indent
  • text-shadow

Pseudo-Elements

You may add these to the end of a selector

  • :first-letter & :first-line
  • :link & :visited
  • :hover & :active

Boxes

You can set properties to control how any box looks

  • width & height
  • min-width & max-width
  • min-height & max-height
  • overflow

Box Appearance

  • border
    • width
    • style
    • color
  • margin
    • centering
  • padding
  • box-shadow
  • border-radius
  • display
  • visibility

The Float!

  • float left aligns your element to the left and brings content up beside it on the right
  • float right aligns your element to the right and brings up content beside it on the left
