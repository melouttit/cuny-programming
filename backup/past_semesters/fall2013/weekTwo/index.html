Meta Tags & SEO

  • Meta tags provide more information about your page to browers
  • They live in the head tag
  • Are useful for SEO purposes as search engines use them to grab more information about your site

Most Common Meta

  • Description: A description of the page and often shown in search results
  • Robots: Indicates whether search engines should add this page to their results.
  • Author: The author of the web page
  • Keywords: A list of words separated by commas that users might search to find your site*
  • You can also specify variables like time updated, a refresh interval, and prevent browers from caching the page, among other things

Meta Examples

  • Description: 
    <meta name="description" content="A website about Meghan Louttit
     and her programming and design work for The New York Times, The Washington Post"     />
  • Robot: 
    <meta name="robots" content="nofollow" />
  • Author: 
    <meta name="author" content="Meghan Elizabeth Louttit" />
  • Keywords: 
    <meta name="keywords" content="Meghan Louttit, portfolio, resume, design,
     web design, The New York Times, The Washington Post, multimedia"     />

Exercise

  • Add each of the four types of meta tags to your homepage
  • Make sure you use relevant keywords and description

Directory Structure

Most websites are organized by folders a.k.a. directories

  • The root folder is the most top-level folder with you content in directories or files within the root.
  • The main homepage of a site or section is always index.html, which is returned when no other file type is specified in the URL
  • For example: http://mysite.com will return the same thing as http://mysite.com/index.html

Example MySite.com

The URL

  • Each image and file on the page has a URL under that site's domain
  • The URL path points to the particular file or directory you want retrieve
  • Find the following in the example site:
    • http://mysite.com/writing links to the Writing page
    • http://mysite.com/about.html links to the About page
    • http://mysite.com/images/artoo.jpg is the path to the artoo.jpg image

Anchor & Deep Links 1

First, identify the different points on your page using IDs:

  • The top of your page: 
    <h1 id="top">How to Code</h1>
  • Various sections: 
    <h2 id="css">What is CSS?</h2>

Anchor & Deep Links 2

To then link to these points, either on your site or from another site:

  • Within your site: 
    <a href="#top">To the Top</a>
  • Linked from another site: 
    <a href="http://mysite.com/#css">Learn CSS from MySite.com!</a>

Containers

  • Inline Elements: Always appear to continue on the same line as their neighboring elements. *Examples include anchors, strong and emphasis, images and span.
  • Block Elements: Ones that always appear to start on a new line in your browser. *Examples include header tags, paragraphs, lists and div.

Grouping Inline: span

  • Use it to contain a section of text that should look different from surrounding text
  • Hold a number of inline elements
  • Needs classes or IDs to be of use to the reader

Grouping Blocks: div

  • Use it to group various elements on your page together
  • Think of the div as a container - you might want it to hold your navigation elements, or all your header elements.
  • Like the span, it usually has a class or ID to make it relevent for positioning and styling

Class & ID

  • Attributes that can be applied to any HTML tag. Used to target elements through CSS.
  • Class: Use it to identify multiple elements within a page 
    <p>There are <span class="important">no subways
     running today</span>, according to <em>The New York Times</em></p>
  • ID: Use it to identify a unique element on a page 
    <div id="navigation">
        <ul><li>Home</li><li>About Me</li></ul>
    </div>

Wireframes

  • This is the next step in the design process, after you've created your Site Map. It is a simple sketch of the information that needs to go on each page.
  • The wireframe should demonstrate the hierarchy of the information and how much space it might require.
  • This the step that connects your site map to your code - you are literally drawing containers that you will then translate into code.
