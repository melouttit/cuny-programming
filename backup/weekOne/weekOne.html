Which line is valid HTML?

  • A. 
    <p><strong>Hello, World!</p></strong>
  • B. 
    <p id="myName'><strong>Hello, World!</strong></p>
  • C. 
    <h1 class='headline'><strong>Hello, World!</strong></p>
  • D. 
    <div class="box"><p>Hello, World!</p>

The following CSS does what?

h1.headline{ 
   color:#000000;
   margin:0 0 10px 0; 
}
  • A. Changes color of all header text with an id of headline to black, and gives it a 10px margin
  • B. Changes color of all header text with a class of headline to black and gives it a bottom margin of 10px
  • C. Changes color of all header text to white and gives it a right margin of 10px
  • D. Changes all elements with class headline to the color white and gives it a bottom margin of 10px

What is HTML?

  • Stands for Hyper-Text Markup Language
  • It is a language for 'writing' or 'describing' webpages. Those pages are then 'read' by your browser.
  • Valid HTML: HTML with the proper syntax. Most browers will interpret some invalid HTML correctly, but it is still not a good idea to leave it that way - your page might not look correct in other browers, older browers, or in newer versions of browsers. Think of it as using bad grammer.
  • Let's set up a page!

Standard Page Setup

<!DOCTYPE html>
<html>

<head>
</head>

<body>
</body>

</html>

DOCTYPE

  • Declared at the beginning of your page
  • Helps browser know how to read your page so it displays correctly
  • Within the doctype you will declare that the page is in HTML and which version you are using
  • There have been a handful of versions of HTML, we'll be sticking with HTML5: 
    <!DOCTYPE html>

Tags

  • The heart of any language markup, even if the syntax is different
  • Each tag is a keyword surrounded by angle brackets: 
    <p>
  • Most tags come in pairs and need to be closed. Closing tags look like this: 
    </p>
  • The basic setup of any HTML content: 
    <tagname>content</tagname>

Attributes

  • HTML tags can take on attributes, which describes the content inside that tag.
  • Attributes are written as key - value pairs: 
    <p align="center">Paragraph text goes here</p>
  • These will come into play later on as we use them in conjunction with CSS to target elements

Add the Following

  • Page title to your <head>
  • Page headline
  • Some paragraph content

How to Diagram

  • Opening the screenshot in Photoshop or a similar editor program and making boxes over the content and using the text tool to note font and color choices
  • In Google Docs: Under the Create New menu, select Drawing. Then, under the Insert menu, select Image. Select your screenshot. Then be sure to select 100% from the View menu to see your image at the correct scale. Use the drawing tools to annotate.
  • Use different colors to note different parts of the page - i.e. headlines, sets of links, naviagtion, etc. Also note common fonts, colors, etc.
